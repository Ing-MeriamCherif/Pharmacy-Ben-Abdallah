💊 Pharmacie Ben Abdallah – Pharmacy Management System

Pharmacie Ben Abdallah is a full-stack desktop pharmacy management system developed as a university project. What began as an academic assignment evolved into a complete real-world solution designed to manage daily pharmacy operations efficiently and intuitively.

The system covers everything from inventory tracking and sales to supplier management and analytics, all within a modern Java Swing desktop application.

🎓 Project Overview

This application is built to simulate and manage the core operations of a pharmacy, including:

Medication inventory control
Client management
Sales processing (POS system)
Purchase and supplier management
Stock monitoring and reporting

It follows a structured MVC architecture to ensure maintainability, scalability, and clean separation of concerns.

🛠️ Tech Stack
Java Swing – Desktop UI framework
MySQL – Relational database
JDBC – Database connectivity
MVC Architecture – Design pattern
Custom UI Design System – Themed interface with enhanced UX
✨ Key Features
📦 Stock & Inventory Management
Full medication catalog management
FEFO (First Expired, First Out) lot tracking
Automatic stock updates
Low-stock and expiry alerts
Supplier-based restocking system
🛒 Sales (POS System)
Fast and intuitive Point of Sale interface
CNAM client lookup integration
Loyalty system for recurring customers
Real-time stock deduction during sales
🚚 Suppliers & Purchases
Supplier database management
Purchase order creation
Automatic stock replenishment after purchases
Tracking of supplier transactions
📊 Analytics & Reporting
Monthly revenue charts
Stock distribution (donut charts)
Export reports in CSV / TXT formats
Sales performance tracking
🎨 UI/UX Design
Custom modern theme
Smooth animated transitions
Clean and user-friendly interface
Consistent design system across modules
🧱 Architecture

The system is structured using the MVC (Model–View–Controller) pattern:

Model → Handles data logic and database operations
View → Java Swing UI components
Controller → Business logic and user interaction handling

This ensures a clean separation between UI and backend logic.

🗄️ Database
MySQL relational database
Fully normalized schema
Tables include:
Medications
Clients
Sales
Purchases
Suppliers
Stock batches (lots)
🚀 Future Improvements
Web-based version (Spring Boot + React)
Barcode scanner integration
Multi-pharmacy support
Advanced AI-based demand forecasting
Cloud database synchronization

Developed by students as part of a university software engineering project.
